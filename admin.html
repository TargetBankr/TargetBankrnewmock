📖 First-time Setup

1️⃣ Create the Google Sheet

Create a sheet with these tabs (exact names): Tests, Config, Questions, Scores.

Headers (row 1) for each tab are listed in the Apps Script file.

2️⃣ Create a Drive folder for images

Create a folder in Google Drive (e.g. "TargetBankr Images"). Right-click → Share → "Anyone with the link" → Viewer. Copy the folder ID from its URL (the long string after /folders/).

3️⃣ Deploy the Apps Script

In your sheet: Extensions → Apps Script. Paste the contents of apps-script.gs, set SPREADSHEET_ID and DRIVE_FOLDER_ID at the top.

⚠ Important — grant Drive permission ONCE: In the function dropdown at the top of the editor, select setupPermissions and click ▶ Run. Approve the Drive permission popup (it will say "Advanced → Go to … (unsafe) → Allow" — that's normal because you wrote the script yourself).

Then Deploy → New deployment → Web app → Execute as: Me → Who has access: Anyone. Copy the Web App URL.

Already deployed and seeing ERR-503? Just run setupPermissions as above, then Deploy → Manage deployments → ✏ → Version: New version → Deploy. Same URL keeps working.

4️⃣ Paste the URL above

Paste your Apps Script URL into the yellow bar above and click "Save & Test". Once it shows "Connected ✓" you're ready.

5️⃣ Publish each sheet tab as CSV

For the student portal: File → Share → Publish to web. Publish each tab as CSV. Paste those URLs into the top of student-portal.html alongside your Firebase config.

6️⃣ Add tests, sections, questions

Use the tabs above. Start with "Add Test", then add Sections for that test, then add Questions for each section. Use "Manage Tests" to edit, preview, or hide tests after creation.

🔥 Firebase Auth Setup

1. Create a Firebase project

Go to console.firebase.google.com → Add project → enable Email/Password authentication.

2. Get your Web App config

Project Settings → General → Your apps → Web → Register app → Copy the firebaseConfig snippet.

3. Paste it into student-portal.html

Open student-portal.html in any text editor. Find the firebaseConfig object near the top and replace the PASTE_… placeholders with your real values.

4. Enable Firestore (for Dashboard + Leaderboard)

Go to Firebase Console → Firestore Database → Create database → Start in test mode → choose your region → Done.
That's it — the student portal will automatically save attempts, populate the leaderboard, and show the dashboard. For production, update your Firestore security rules to require auth.

5. (Optional) Quick-create student accounts

Students can self-register in the student portal. If you need to bulk-create accounts, use Firebase Console → Authentication → Users → Add user.

🆘 Error Code Reference

If a student or admin sees an error code, look it up here. The code is shown to the student instead of a confusing technical message.

CodeMeaningHow to fix
ERR-101Connection failedCheck that Sheet CSV URLs are correct and published. Check internet.
ERR-102Test config missingThe selected test has no row in the Tests sheet.
ERR-103No questions availableAdd at least one question for the test in the Questions sheet.
ERR-104Authentication failedFirebase config is missing/wrong, or sign-in failed. Check Firebase keys.
ERR-105Score save failedApps Script URL not set or rejected the request. Score is shown locally.
ERR-106No tests in this categoryMark at least one test "active" with this category.
ERR-107No sections configuredAdd at least one Config row for the test.
ERR-411Test ID already existsPick a different Test ID when adding.
ERR-412Test not foundThe test was deleted or the ID is wrong.
ERR-422Section not foundThe Test ID + Section ID combo doesn't exist in Config.
ERR-432Question not foundThe Test+Section+QNo doesn't exist in Questions.
ERR-501Drive folder not setSet DRIVE_FOLDER_ID at the top of apps-script.gs and redeploy.
ERR-502No file dataImage upload failed before sending; try again.
ERR-503Drive permission not grantedIn the Apps Script editor, select the setupPermissions function from the dropdown, click Run, and approve Drive access. Then Deploy → Manage deployments → ✏ → Version: New version → Deploy.
ERR-504Drive folder not foundThe DRIVE_FOLDER_ID is wrong or the folder was deleted. Copy the long string after /folders/ in the folder URL.
ERR-900Server errorApps Script threw an exception. Check the Apps Script execution log.
ERR-901Unknown actionUpdate the Apps Script to the latest version.
